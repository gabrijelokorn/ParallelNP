8 """
9 deepcopy(x)
10
11 Create a deep copy of `x`: everything is copied recursively, resulting in a fully
12 independent object. For example, deep-copying an array creates deep copies of all
13 the objects it contains and produces a new array with the consistent relationship
14 structure (e.g., if the first two elements are the same object in the original array,
15 the first two elements of the new array will also be the same `deepcopy`ed object).
16 Calling `deepcopy` on an object should generally
17 have the same effect as serializing and then deserializing it.
18
19 While it isn't normally necessary, user-defined types can override the default `deepcopy`
20 behavior by defining a specialized version of the function
21 `deepcopy_internal(x::T, dict::IdDict)` (which shouldn't otherwise be used),
22 where `T` is the type to be specialized for, and `dict` keeps track of objects copied
23 so far within the recursion. Within the definition, `deepcopy_internal` should be used
24 in place of `deepcopy`, and the `dict` variable should be
25 updated as appropriate before returning.
26 """
