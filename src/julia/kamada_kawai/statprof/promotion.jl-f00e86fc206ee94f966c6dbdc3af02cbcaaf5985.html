259 """
260 promote_type(type1, type2, ...)
261
262 Promotion refers to converting values of mixed types to a single common type.
263 `promote_type` represents the default promotion behavior in Julia when
264 operators (usually mathematical) are given arguments of differing types.
265 `promote_type` generally tries to return a type which can at least approximate
266 most values of either input type without excessively widening. Some loss is
267 tolerated; for example, `promote_type(Int64, Float64)` returns
268 [`Float64`](@ref) even though strictly, not all [`Int64`](@ref) values can be
269 represented exactly as `Float64` values.
270
271 See also: [`promote`](@ref), [`promote_typejoin`](@ref), [`promote_rule`](@ref).
272

508 """
509 promote_op(f, argtypes...)
510
511 Guess what an appropriate container eltype would be for storing results of
512 `f(::argtypes...)`. The guess is in part based on type inference, so can change any time.
513
514 Accordingly, return a type `R` such that `f(args...) isa R` where `args isa T`.
515
516 !!! warning
517 Due to its fragility, use of `promote_op` should be avoided. It is preferable to base
518 the container eltype on the type of the actual elements. Only in the absence of any
519 elements (for an empty result container), it may be unavoidable to call `promote_op`.
520
521 The type `R` obtained from `promote_op` is merely an upper bound. There may exist a stricter
522 type `S` such that `f(args...) isa S` for every `args isa T` with `S <: R` and `S != R`.
523 Furthermore, the exact type `R` obtained from `promote_op` depends on various factors
524 including but not limited to the exact Julia version used, packages loaded, and command line
525 options. As such, when used in publicly registered packages, **it is the package authors'
526 responsibility to ensure that the API guarantees provided by the package do not depend on
527 the exact type `R` obtained from `promote_op`.**
528
529 Additionally, the result may return overly exact types, such as `DataType`, `Type`, or
530 `Union{...}`, while the desired inputs or outputs may be different from those. The internal
531 `promote_typejoin_union` function may be helpful to improve the result in some of these
532 cases.
533
534 # Extended help
535
536 ## Examples
537
538 The following function is an invalid use-case of `promote_op`.
539
540 ```julia
541 \"""
542 invalid_usecase1(f, xs::AbstractArray) -> ys::Array
543
544 Return an array `ys` such that `vec(ys)` is `isequal`-equivalent to
545
546 [f(xs[1]), f(xs[2]), ..., f(xs[end])]
547 \"""
548 function invalid_usecase1(f, xs)
549 R = promote_op(f, eltype(xs))
550 ys = similar(xs, R)
551 for i in eachindex(xs, ys)
552 ys[i] = f(xs[i])
553 end
554 return ys
555 end
556 ```
557
558 This is because the value obtained through `eltype(invalid_usecase1(f, xs))` depends on
559 exactly what `promote_op` returns. It may be improved by re-computing the element type
560 before returning the result.
561
562 ```julia
563 function valid_usecase1(f, xs)
564 R = promote_typejoin_union(promote_op(f, eltype(xs)))
565 ys = similar(xs, R)
566 S = Union{}
567 for i in eachindex(xs, ys)
568 ys[i] = f(xs[i])
569 S = promote_type(S, typeof(ys[i]))
570 end
571 if S != R
572 zs = similar(xs, S)
573 copyto!(zs, ys)
574 return zs
575 end
576 return ys
577 end
578 ```
579
580 Note that using [`isconcretetype`](@ref) on the result is not enough to safely use
581 `promote_op`. The following function is another invalid use-case of `promote_op`.
582
583 ```julia
584 function invalid_usecase2(f, xs)
585 R = promote_op(f, eltype(xs))
586 if isconcretetype(R)
587 ys = similar(xs, R)
588 else
589 ys = similar(xs, Any)
590 end
591 for i in eachindex(xs, ys)
592 ys[i] = f(xs[i])
593 end
594 return ys
595 end
596 ```
597
598 This is because whether or not the caller gets `Any` element type depends on if `promote_op`
599 can infer a concrete return type of the given function. A fix similar to `valid_usecase1`
600 can be used.
601
602 *Technically*, another possible fix for `invalid_usecase1` and `invalid_usecase2` is to
603 loosen the API guarantee:
604
605 > another_valid_usecase1(f, xs::AbstractArray) -> ys::Array
606 >
607 > Return an array `ys` such that every element in `xs` with the same index
608 > is mapped with `f`.
609 >
610 > The element type of `ys` is _undefined_. It must not be used with generic
611 > functions whose behavior depend on the element type of `ys`.
612
613 However, it is discouraged to define such unconventional API guarantees.
614 """
615 function promote_op(f, S::Type...)
616 argT = TupleOrBottom(S...)
617 argT === Union{} && return Union{}
618 return _return_type(f, argT)
619 end
