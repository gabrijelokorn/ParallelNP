18 Recursive use will result in a deadlock.
19 This kind of lock should only be used around code that takes little time
20 to execute and does not block (e.g. perform I/O).
21 In general, [`ReentrantLock`](@ref) should be used instead.
22
23 Each [`lock`](@ref) must be matched with an [`unlock`](@ref).
24 If [`!islocked(lck::SpinLock)`](@ref islocked) holds, [`trylock(lck)`](@ref trylock)
25 succeeds unless there are other tasks attempting to hold the lock "at the same time."
26
27 Test-and-test-and-set spin locks are quickest up to about 30ish
28 contending threads. If you have more contention than that, different
