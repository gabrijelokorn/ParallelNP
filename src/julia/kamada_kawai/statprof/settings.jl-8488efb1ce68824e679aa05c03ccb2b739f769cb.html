137 # ArgParseSettings
138 """
139 ArgParseSettings
140
141 The `ArgParseSettings` object contains all the settings to be used during argument parsing. Settings
142 are divided in two groups: general settings and argument-table-related settings.
143 While the argument table requires specialized functions such as [`@add_arg_table!`](@ref) to be
144 defined and manipulated, general settings are simply object fields (most of them are `Bool` or
145 `String`) and can be passed to the constructor as keyword arguments, or directly set at any time.
146
147 This is the list of general settings currently available:
148
149 * `prog` (default = `""`): the name of the program, as displayed in the auto-generated help and
150 usage screens. If left empty, the source file name will be used.
151 * `description` (default = `""`): a description of what the program does, to be displayed in the
152 auto-generated help-screen, between the usage lines and the arguments description. If
153 `preformatted_description` is `false` (see below), it will be automatically formatted, but you can
154 still force newlines by using two consecutive newlines in the string, and manually control spaces
155 by using non-breakable spaces (the character `'\\ua0'`).
156 * `preformatted_description` (default = `false`): disable automatic formatting of `description`.
157 * `epilog` (default = `""`): like `description`, but will be displayed at the end of the
158 help-screen, after the arguments description. The same formatting rules also apply.
159 * `preformatted_epilog` (default = `false`): disable automatic formatting of `epilog`.
160 * `usage` (default = `""`): the usage line(s) to be displayed in the help screen and when an error
161 is found during parsing. If left empty, it will be auto-generated.
162 * `version` (default = `"Unknown version"`): version information. It's used by the `:show_version`
163 action.
164 * `add_help` (default = `true`): if `true`, a `--help, -h` option (triggering the `:show_help`
165 action) is added to the argument table.
166 * `add_version` (default = `false`): if `true`, a `--version` option (triggering the `:show_version`
167 action) is added to the argument table.
168 * `help_width` (default = `70`): set the width of the help text. This does not affect the
169 `usage` line if it is provided by the user rather than being auto-generated; it also does not
170 affect the `description/epilog` lines if the `preformatted_description/epilog` options are set to
171 `false`.
172 * `help_alignment_width` (default = `24`): set the maximum width of the left column of the help
173 text, where options and arguments names are listed. This also affects the indentation of the usage
174 line when it is auto-generated. It must be ≥ 4 and should be less than `help_width`.
175 * `fromfile_prefix_chars` (default = `Set{Char}()`): an argument beginning with one of these
176 characters will specify a file from which arguments will be read, one argument read per line.
177 Alphanumeric characters and the hyphen-minus (`'-'`) are prohibited.
178 * `autofix_names` (default = `false`): if `true`, will try to automatically fix the uses of dashes
179 (`'-'`) and underscores (`'_'`) in option names and destinations: all underscores will be
180 converted to dashes in long option names; also, associated destination names, if auto-generated
181 (see the [Argument names](@ref) section), will have dashes replaced with underscores, both for
182 long options and for positional arguments. For example, an option declared as `"--my-opt"` will be
183 associated with the key `"my_opt"` by default. It is especially advisable to turn this option on
184 then parsing with the `as_symbols=true` argument to `parse_args`.
185 * `error_on_conflict` (default = `true`): if `true`, throw an error in case conflicting entries are
186 added to the argument table; if `false`, later entries will silently take precedence. See the
187 [Conflicts and overrides](@ref) srction for a detailed description of what conflicts are and what
188 is the exact behavior when this setting is `false`.
189 * `suppress_warnings` (default = `false`): if `true`, all warnings will be suppressed.
190 * `allow_ambiguous_opts` (default = `false`): if `true`, ambiguous options such as `-1` will be
191 accepted.
192 * `commands_are_required` (default = `true`): if `true`, commands will be mandatory. See the
193 [Commands](@ref) section.
194 * `exc_handler` (default = `ArgParse.default_handler`): this is a function which is invoked when an
195 error is detected during parsing (e.g. an option is not recognized, a required argument is not
196 passed etc.). It takes two arguments: the `settings::ArgParseSettings` object and the
197 `err::ArgParseError` exception. The default handler behaves differently depending on whether it's
198 invoked from a script or in an interactive environment (e.g. REPL/IJulia). In non-interactive
199 (script) mode, it calls `ArgParse.cmdline_handler`, which prints the error text and the usage
200 screen on standard error and exits Julia with error code 1:
201
202 ```julia
203 function cmdline_handler(settings::ArgParseSettings, err, err_code::Int = 1)
204 println(stderr, err.text)
205 println(stderr, usage_string(settings))
206 exit(err_code)
207 end
208 ```
209
210 In interactive mode instead it calls the function `ArgParse.debug_handler`, which just rethrows
211 the error.
212 * `exit_after_help` (default = `!isinteractive()`): exit Julia (with error code `0`) when the
213 `:show_help` or `:show_version` actions are triggered. If `false`, those actions will just stop
214 the parsing and make `parse_args` return `nothing`.
215
216 Here is a usage example:
217
218 ```julia
219 settings = ArgParseSettings(description = "This program does something",
220 commands_are_required = false,
221 version = "1.0",
222 add_version = true)
223 ```
224
225 which is also equivalent to:
226
227 ```julia
228 settings = ArgParseSettings()
229 settings.description = "This program does something."
230 settings.commands_are_required = false
231 settings.version = "1.0"
232 settings.add_version = true
233 ```
234
235 As a shorthand, the `description` field can be passed without keyword, which makes this equivalent
236 to the above:
237
238 ```julia
239 settings = ArgParseSettings("This program does something",
240 commands_are_required = false,
241 version = "1.0",
242 add_version = true)
243 ```
244
245 Most settings won't take effect until `parse_args` is invoked, but a few will have immediate
246 effects: `autofix_names`, `error_on_conflict`, `suppress_warnings`, `allow_ambiguous_opts`.
247 """

651 end

686 """
687 add_arg_table!(settings, [arg_name [,arg_options]]...)
688
689 This function is very similar to the macro version [`@add_arg_table!`](@ref). Its syntax is stricter:
690 tuples and blocks are not allowed and argument options are explicitly specified as `Dict` objects.
691 However, since it doesn't involve macros, it offers more flexibility in other respects, e.g. the
692 `arg_name` entries need not be explicit, they can be anything which evaluates to a `String` or a
693 `Vector{String}`.
694
695 Example:
696
697 ```julia
698 add_arg_table!(settings,
699 ["--opt1", "-o"],
700 Dict(
701 :help => "an option with an argument"
702 ),
703 "--opt2",
704 "arg1",
705 Dict(
706 :help => "a positional argument"
707 :required => true
708 ))
709 ```
710 """
711 function add_arg_table!(settings::ArgParseSettings, table::Union{ArgName,Vector,Dict}...)
712 has_name = false
713 for i = 1:length(table)
714 !has_name && !(table[i] isa ArgName) &&
715 serror("option field must be preceded by the arg name")
716 has_name = true
717 end
718 i = 1
719 while i ≤ length(table)
720 if i+1 ≤ length(table) && !(table[i+1] isa ArgName)
721 add_arg_field!(settings, table[i]; table[i+1]...)
722 i += 2
723 else
724 add_arg_field!(settings, table[i])
725 i += 1
726 end
727 end
728 return settings
729 end

731 """
732 @add_arg_table!(settings, table...)
733
734 This macro adds a table of arguments and options to the given `settings`. It can be invoked multiple
735 times. The arguments groups are determined automatically, or the current default group is used if
736 specified (see the [Argument groups](@ref) section for more details).
737
738 The `table` is a list in which each element can be either `String`, or a tuple or a vector of
739 `String`, or an assigmment expression, or a block:
740
741 * a `String`, a tuple or a vector introduces a new positional argument or option. Tuples and vectors
742 are only allowed for options or commands, and provide alternative names (e.g. `["--opt", "-o"]` or
743 `["checkout", "co"]`)
744 * assignment expressions (i.e. expressions using `=`, `:=` or `=>`) describe the previous argument
745 behavior (e.g. `help = "an option"` or `required => false`). See the
746 [Argument entry settings](@ref) section for a complete description
747 * blocks (`begin...end` or lists of expressions in parentheses separated by semicolons) are useful
748 to group entries and span multiple lines.
749
750 These rules allow for a variety usage styles, which are discussed in the
751 [Argument table styles](@ref) section. In the rest of the documentation, we will mostly use this
752 style:
753
754 ```julia
755 @add_arg_table! settings begin
756 "--opt1", "-o"
757 help = "an option with an argument"
758 "--opt2"
759 "arg1"
760 help = "a positional argument"
761 required = true
762 end
763 ```
764
765 In the above example, the `table` is put in a single `begin...end` block and the line
766 `"--opt1", "-o"` is parsed as a tuple; indentation is used to help readability.
767
768 See also the function [`add_arg_table!`](@ref).
769 """
770 macro add_arg_table!(s, x...)
771 _add_arg_table!(s, x...)
772 end
773
774 # Moved all the code to a function just to make the deprecation work
775 function _add_arg_table!(s, x...)
776 # transform the tuple into a vector, so that
777 # we can manipulate it
778 x = Any[x...]
779 # escape the ArgParseSettings
780 s = esc(s)
781 z = esc(gensym())
782 # start building the return expression
783 exret = quote
784 $z = $s
785 $z isa ArgParseSettings ||
786 serror("first argument to @add_arg_table! must be of type ArgParseSettings")
787 end
788 # initialize the name and the options expression
789 name = nothing
790 exopt = Any[:Dict]
791
792 # iterate over the arguments
793 i = 1
794 while i ≤ length(x)
795 y = x[i]
796 if Meta.isexpr(y, :block)
797 # found a begin..end block: expand its contents
798 # in-place and restart from the same position
799 splice!(x, i, y.args)
800 continue
801 elseif Meta.isexpr(y, :macrocall) &&
802 ((y.args[1] == GlobalRef(Core, Symbol("@doc"))) ||
803 (Meta.isexpr(y.args[1], :core) && y.args[1].args[1] == Symbol("@doc")))
804 # Was parsed as doc syntax. Split into components
805 splice!(x, i, y.args[2:end])
806 continue
807 elseif (y isa AbstractString) || Meta.isexpr(y, (:vect, :tuple))
808 Meta.isexpr(y, :tuple) && (y.head = :vect) # transform tuples into vectors
809 if Meta.isexpr(y, :vect) && (isempty(y.args) || !all(x->x isa AbstractString, y.args))
810 # heterogeneous elements: splice it in place, just like blocks
811 splice!(x, i, y.args)
812 continue
813 end
814 # found a string, or a vector/tuple of strings:
815 # this must be the option name
816 if name ≢ nothing
817 # there was a previous arg field on hold
818 # first, concretely build the options
819 opt = Expr(:call, exopt...)
820 kopts = Expr(:parameters, Expr(:(...), opt))
821 # then, call add_arg_field!
822 aaf = Expr(:call, :add_arg_field!, kopts, z, name)
823 # store it in the output expression
824 exret = quote
825 $exret
826 330 (1 %) 384 (1 %)
384 (1 %) samples spent in macro expansion
330 (86 %) (ex.), 384 (100 %) (incl.) when called from main line 22
44 (81 %) samples spent calling typeinf_ext_toplevel
10 (19 %) samples spent calling Dict
$aaf
827 end
828 end
829 # put the name on hold, reinitialize the options expression
830 name = y
831 exopt = Any[:Dict]
832 i += 1
833 elseif Meta.isexpr(y, (:(=), :(:=), :kw))
834 # found an assignment: add it to the current options expression
835 name ≢ nothing ||
836 serror("malformed table: description fields must be preceded by the arg name")
837 push!(exopt, Expr(:call, :(=>), Expr(:quote, y.args[1]), esc(y.args[2])))
838 i += 1
839 elseif Meta.isexpr(y, :call) && y.args[1] == :(=>)
840 # found an assignment: add it to the current options expression
841 name ≢ nothing ||
842 serror("malformed table: description fields must be preceded by the arg name")
843 push!(exopt, Expr(:call, :(=>), Expr(:quote, y.args[2]), esc(y.args[3])))
844 i += 1
845 elseif (y isa LineNumberNode) || Meta.isexpr(y, :line)
846 # a line number node, ignore
847 i += 1
848 continue
849 else
850 # anything else: ignore, but issue a warning
851 @warn "@add_arg_table!: ignoring expression $y"
852 i += 1
853 end
854 end
855 if name ≢ nothing
856 # there is an arg field on hold
857 # same as above
858 opt = Expr(:call, exopt...)
859 kopts = Expr(:parameters, Expr(:(...), opt))
860 aaf = Expr(:call, :add_arg_field!, kopts, z, name)
861 exret = quote
862 $exret
863 $aaf
864 end
865 end
866
867 # the return value when invoking the macro
868 # will be the ArgParseSettings object
869 exret = quote
870 $exret
871 $z
872 end
873
874 # return the resulting expression
875 exret
876 end

894
895 supplied_opts = keys(desc)
896
897 @defaults desc begin
898 nargs = ArgConsumerType()
899 action = default_action(nargs)
900 arg_type = Any
901 default = nothing
902 constant = nothing
903 required = false
904 range_tester = x->true
905 eval_arg = false
906 dest_name = ""
907 help = ""
908 metavar = ""
909 force_override = !settings.error_on_conflict
910 group = settings.default_group
911 end
912
913 check_type(nargs, Union{ArgConsumerType,Int,Char}, "nargs must be an Int or a Char")
914 check_type(action, Union{AbstractString,Symbol}, "action must be an AbstractString or a Symbol")
915 check_type(arg_type, Type, "invalid arg_type")
916 check_type(required, Bool, "required must be a Bool")
917 check_type(range_tester, Function, "range_tester must be a Function")
918 check_type(dest_name, AbstractString, "dest_name must be an AbstractString")
919 check_type(help, AbstractString, "help must be an AbstractString")
920 # Check metavar's type to be either an AbstractString or a
921 # Vector{T<:AbstractString}
922 metavar_error = "metavar must be an AbstractString or a Vector{<:AbstractString}"
923 if !(metavar isa AbstractString)
924 check_type(metavar, Vector, metavar_error)
925 check_eltype(metavar, AbstractString, metavar_error)
926 check_type(nargs, Integer, "nargs must be an integer for multiple metavars")
927 length(metavar) == nargs || serror("metavar array must have length of nargs")
928 end
929 check_type(force_override, Bool, "force_override must be a Bool")
930 check_type(group, Union{AbstractString,Symbol}, "group must be an AbstractString or a Symbol")
931
932 nargs isa ArgConsumerType || (nargs = ArgConsumerType(nargs))
933 action isa Symbol || (action = Symbol(action))
934
935 is_opt = name isa Vector ?
936 startswith(first(name), '-') :
937 startswith(name, '-')
938
939 check_action_is_valid(action)
940
941 action == :command && (action = is_opt ? :command_flag : :command_arg)
942
943 check_nargs_and_action(nargs, action)
944
945 new_arg = ArgParseField()
946
947 is_flag = is_flag_action(action)
948
949 if !is_opt
950 is_flag && serror("invalid action for positional argument: $action")
951 nargs.desc == :? && serror("invalid 'nargs' for positional argument: '?'")
952 metavar isa Vector && serror("multiple metavars only supported for optional arguments")
953 end
954
955 pos_arg, long_opts, short_opts, cmd_aliases = name_to_fieldnames!(settings, name)
956
957 if !isempty(cmd_aliases)
958 is_command_action(action) || serror("only command arguments can have multiple names (aliases)")
959 end
960
961 new_arg.dest_name = auto_dest_name(pos_arg, long_opts, short_opts, settings.autofix_names)
962
963 new_arg.long_opt_name = long_opts
964 new_arg.short_opt_name = short_opts
965 new_arg.cmd_aliases = cmd_aliases
966 new_arg.nargs = nargs
967 new_arg.action = action
968
969 group = string(group)
970 if :group ∈ supplied_opts && !isempty(group)
971 check_group_name(group)
972 end
973 arg_group = get_group(group, new_arg, settings)
974 new_arg.group = arg_group.name
975 if arg_group.exclusive && (!is_opt || is_command_action(action))
976 serror("group $(new_arg.group) is mutually-exclusive, actions and commands are not allowed")
977 end
978
979 if action ∈ (:store_const, :append_const) && :constant ∉ supplied_opts
980 serror("action $action requires the 'constant' field")
981 end
982
983 valid_keys = [:nargs, :action, :help, :force_override, :group]
984 if is_flag
985 if action ∈ (:store_const, :append_const)
986 append!(valid_keys, [:default, :constant, :arg_type, :dest_name])
987 elseif action ∈ (:store_true, :store_false, :count_invocations, :command_flag)
988 push!(valid_keys, :dest_name)
989 else
990 action ∈ (:show_help, :show_version) || found_a_bug()
991 end
992 elseif is_opt
993 append!(valid_keys,
994 [:arg_type, :default, :range_tester, :dest_name, :required, :metavar, :eval_arg])
995 nargs.desc == :? && push!(valid_keys, :constant)
996 elseif action ≠ :command_arg
997 append!(valid_keys, [:arg_type, :default, :range_tester, :required, :metavar])
998 end
999 settings.suppress_warnings || warn_extra_opts(supplied_opts, valid_keys)
1000
1001 if is_command_action(action)
1002 if (:dest_name ∈ supplied_opts) && (:dest_name ∈ valid_keys)
1003 cmd_name = dest_name
1004 else
1005 cmd_name = new_arg.dest_name
1006 end
1007 end
1008 if (:dest_name ∈ supplied_opts) && (:dest_name ∈ valid_keys) && (action ≠ :command_flag)
1009 new_arg.dest_name = dest_name
1010 end
1011
1012 check_dest_name(dest_name)
1013
1014 set_if_valid(k, x) = k ∈ valid_keys && setfield!(new_arg, k, x)
1015
1016 set_if_valid(:arg_type, arg_type)
1017 set_if_valid(:default, deepcopy(default))
1018 set_if_valid(:constant, deepcopy(constant))
1019 set_if_valid(:range_tester, range_tester)
1020 set_if_valid(:required, required)
1021 set_if_valid(:help, help)
1022 set_if_valid(:metavar, metavar)
1023 set_if_valid(:eval_arg, eval_arg)
1024
1025 if !is_flag
1026 isempty(new_arg.metavar) && (new_arg.metavar = auto_metavar(new_arg.dest_name, is_opt))
1027 check_metavar(new_arg.metavar)
1028 end
1029
1030 if is_command_action(action)
1031 new_arg.dest_name = cmd_dest_name
1032 new_arg.arg_type = AbstractString
1033 new_arg.constant = cmd_name
1034 new_arg.metavar = cmd_name
1035 cmd_prog_hint = get_cmd_prog_hint(new_arg)
1036 end
1037
1038 if is_flag
1039 if action == :store_true
1040 new_arg.arg_type = Bool
1041 new_arg.default = false
1042 new_arg.constant = true
1043 elseif action == :store_false
1044 new_arg.arg_type = Bool
1045 new_arg.default = true
1046 new_arg.constant = false
1047 elseif action == :count_invocations
1048 new_arg.arg_type = Int
1049 new_arg.default = 0
1050 elseif action == :store_const
1051 check_default_type(new_arg.default, new_arg.arg_type)
1052 check_default_type(new_arg.constant, new_arg.arg_type)
1053 new_arg.default = convert_default(new_arg.arg_type, new_arg.default)
1054 elseif action == :append_const
1055 check_default_type(new_arg.constant, new_arg.arg_type)
1056 if :arg_type ∉ supplied_opts
1057 new_arg.arg_type = typeof(new_arg.constant)
1058 end
1059 check_default_type_multi(new_arg.default, new_arg.arg_type)
1060 new_arg.default = convert_default_multi(new_arg.arg_type, new_arg.default)
1061 elseif action == :command_flag
1062 # nothing to do
1063 elseif action == :show_help || action == :show_version
1064 # nothing to do
1065 else
1066 found_a_bug()
1067 end
1068 else
1069 arg_type = new_arg.arg_type
1070 range_tester = new_arg.range_tester
1071 default = new_arg.default
1072
1073 if !is_multi_action(new_arg.action) && !is_multi_nargs(new_arg.nargs)
1074 check_default_type(default, arg_type)
1075 check_range_default(default, range_tester)
1076 new_arg.default = convert_default(arg_type, default)
1077 elseif !is_multi_action(new_arg.action) || !is_multi_nargs(new_arg.nargs)
1078 check_default_type_multi(default, arg_type)
1079 check_range_default_multi(default, range_tester)
1080 new_arg.default = convert_default_multi(arg_type, default)
1081 else
1082 check_default_type_multi2(default, arg_type)
1083 check_range_default_multi2(default, range_tester)
1084 new_arg.default = convert_default_multi2(arg_type, default)
1085 end
1086
1087 if is_opt && nargs.desc == :?
1088 constant = new_arg.constant
1089 check_default_type(constant, arg_type)
1090 check_range_default(constant, range_tester)
1091 new_arg.constant = convert_default(arg_type, constant)
1092 end
1093 end
1094
1095 if action == :command_arg
1096 for f in settings.args_table.fields
1097 if f.action == :command_arg
1098 new_arg.fake = true
1099 break
1100 end
1101 end
1102 end
1103
1104 check_arg_makes_sense(settings, new_arg)
1105
1106 check_conflicts_with_commands(settings, new_arg, false)
1107 if force_override
1108 override_duplicates!(settings.args_table.fields, new_arg)
1109 else
1110 check_for_duplicates(settings.args_table.fields, new_arg)
1111 end
1112 push!(settings.args_table.fields, new_arg)
1113 is_command_action(action) && add_command!(settings, cmd_name, cmd_prog_hint, force_override)
1114 return
1115 end

1157 add_arg_group!(settings, description, [name , [set_as_default]]; keywords...)
1158
1159 This function adds an argument group to the argument table in `settings`. The `description` is a
1160 `String` used in the help screen as a title for that group. The `name` is a unique name which can be
1161 provided to refer to that group at a later time.
1162
1163 Groups can be declared to be mutually exclusive and/or required, see below.
1164
1165 After invoking this function, all subsequent invocations of the [`@add_arg_table!`](@ref) macro and
1166 [`add_arg_table!`](@ref) function will use the new group as the default, unless `set_as_default` is
1167 set to `false` (the default is `true`, and the option can only be set if providing a `name`).
1168 Therefore, the most obvious usage pattern is: for each group, add it and populate the argument
1169 table of that group. Example:
1170
1171 ```julia-repl
1172 julia> settings = ArgParseSettings();
1173
1174 julia> add_arg_group!(settings, "custom group");
1175
1176 julia> @add_arg_table! settings begin
1177 "--opt"
1178 "arg"
1179 end;
1180
1181 julia> parse_args(["--help"], settings)
1182 usage: <command> [--opt OPT] [-h] [arg]
1183
1184 optional arguments:
1185 -h, --help show this help message and exit
1186
1187 custom group:
1188 --opt OPT
1189 arg
1190 ```
1191
1192 As seen from the example, new groups are always added at the end of existing ones.
1193
1194 The `name` can also be passed as a `Symbol`. Forbidden names are the standard groups names
1195 (`"command"`, `"positional"` and `"optional"`) and those beginning with a hash character `'#'`.
1196
1197 In order to declare a group as mutually exclusive, use the keyword `exclusive = true`. Mutually
1198 exclusive groups can only contain options, not arguments nor commands, and parsing will fail if more
1199 than one option from the group is provided.
1200
1201 A group can be declared as required using the `required = true` keyword, in which case at least one
1202 option or positional argument or command from the group must be provided.
1203 """
1204 function add_arg_group!(settings::ArgParseSettings,
1205 desc::AbstractString,
1206 tag::Union{AbstractString,Symbol},
1207 set_as_default::Bool = true;
1208 exclusive::Bool = false,
1209 required::Bool = false
1210 )
1211 name = string(tag)
1212 check_group_name(name)
1213 _add_arg_group!(settings, desc, name, set_as_default, exclusive, required)
1214 end

1230 set_default_arg_group!(settings, [name])
1231
1232 Set the default group for subsequent invocations of the [`@add_arg_table!`](@ref) macro and
1233 [`add_arg_table!`](@ref) function. `name` is a `String`, and must be one of the standard group names
1234 (`"command"`, `"positional"` or `"optional"`) or one of the user-defined names given in
1235 `add_arg_group!` (groups with no assigned name cannot be used with this function).
1236
1237 If `name` is not provided or is the empty string `""`, then the default behavior is reset (i.e.
1238 arguments will be automatically assigned to the standard groups). The `name` can also be passed as a
1239 `Symbol`.
1240 """
1241 function set_default_arg_group!(settings::ArgParseSettings, name::Union{AbstractString,Symbol} = "")
1242 name = string(name)
1243 startswith(name, '#') && serror("invalid group name: $name (begins with #)")
1244 isempty(name) && (settings.default_group = ""; return)
1245 found = any(ag->ag.name==name, settings.args_groups)
1246 found || serror("group $name not found")
1247 settings.default_group = name
1248 return
1249 end

1252 function override_conflicts_with_commands!(settings::ArgParseSettings, new_cmd::AbstractString)
1253 ids0 = Int[]
1254 for ia in 1:length(settings.args_table.fields)
1255 a = settings.args_table.fields[ia]
1256 new_cmd == a.dest_name && push!(ids0, ia)
1257 end
1258 while !isempty(ids0)
1259 splice!(settings.args_table.fields, pop!(ids0))
1260 end
1261 end
1262 function override_duplicates!(args::Vector{ArgParseField}, new_arg::ArgParseField)
1263 ids0 = Int[]
1264 for (ia,a) in enumerate(args)
1265 if (a.dest_name == new_arg.dest_name) &&
1266 ((a.arg_type ≠ new_arg.arg_type) ||
1267 (is_multi_action(a.action) && !is_multi_action(new_arg.action)) ||
1268 (!is_multi_action(a.action) && is_multi_action(new_arg.action)))
1269 # unsolvable conflict, mark for deletion
1270 push!(ids0, ia)
1271 continue
1272 end
1273 if is_arg(a) && is_arg(new_arg) && !(is_cmd(a) && is_cmd(new_arg)) &&
1274 a.metavar == new_arg.metavar
1275 # unsolvable conflict, mark for deletion
1276 push!(ids0, ia)
1277 continue
1278 end
1279
1280 # delete conflicting command aliases for different commands
1281 if is_cmd(a) && is_cmd(new_arg) && a.constant ≠ new_arg.constant
1282 ids = Int[]
1283 for (ial1, al1) in enumerate(a.cmd_aliases)
1284 if al1 == new_arg.constant
1285 push!(ids, ial1)
1286 else
1287 for al2 in new_arg.cmd_aliases
1288 al1 == al2 && push!(ids, ial1)
1289 end
1290 end
1291 end
1292 while !isempty(ids)
1293 splice!(a.cmd_aliases, pop!(ids))
1294 end
1295 end
1296
1297 if is_arg(a) || is_arg(new_arg)
1298 # not an option, skip
1299 continue
1300 end
1301
1302 if is_cmd(a) && is_cmd(new_arg) && a.constant == new_arg.constant && !is_arg(a)
1303 is_arg(new_arg) && found_a_bug() # this is ensured by check_settings_are_compatible
1304 # two command flags with the same command -> should have already been taken care of,
1305 # by either check_settings_are_compatible or merge_commands!
1306 continue
1307 end
1308
1309 # delete conflicting long options
1310 ids = Int[]
1311 for il1 = 1:length(a.long_opt_name), l2 in new_arg.long_opt_name
1312 l1 = a.long_opt_name[il1]
1313 l1 == l2 && push!(ids, il1)
1314 end
1315 while !isempty(ids)
1316 splice!(a.long_opt_name, pop!(ids))
1317 end
1318
1319 # delete conflicting short options
1320 ids = Int[]
1321 for is1 in 1:length(a.short_opt_name), s2 in new_arg.short_opt_name
1322 s1 = a.short_opt_name[is1]
1323 s1 == s2 && push!(ids, is1)
1324 end
1325 while !isempty(ids)
1326 splice!(a.short_opt_name, pop!(ids))
1327 end
1328
1329 # if everything was deleted, remove the field altogether
1330 # (i.e. mark it for deletion)
1331 isempty(a.long_opt_name) && isempty(a.short_opt_name) && push!(ids0, ia)
1332 end
1333
1334 # actually remove the marked fields
1335 while !isempty(ids0)
1336 splice!(args, pop!(ids0))
1337 end
1338 end

1340 function check_settings_are_compatible(settings::ArgParseSettings, other::ArgParseSettings)
1341 table = settings.args_table
1342 otable = other.args_table
1343
1344 for a in otable.fields
1345 check_conflicts_with_commands(settings, a, true)
1346 settings.error_on_conflict && check_for_duplicates(table.fields, a)
1347 end
1348
1349 for (subk, subs) in otable.subsettings
1350 settings.error_on_conflict && check_conflicts_with_commands(settings, subk)
1351 haskey(settings, subk) && check_settings_are_compatible(settings[subk], subs)
1352 end
1353 return true
1354 end

1356 function merge_commands!(fields::Vector{ArgParseField}, ofields::Vector{ArgParseField})
1357 oids = Int[]
1358 for a in fields, ioa = 1:length(ofields)
1359 oa = ofields[ioa]
1360 if is_cmd(a) && is_cmd(oa) && a.constant == oa.constant
1361 is_arg(a) ≠ is_arg(oa) && found_a_bug() # ensured by check_settings_are_compatible
1362 for l in oa.long_opt_name
1363 l ∈ a.long_opt_name || push!(a.long_opt_name, l)
1364 end
1365 for s in oa.short_opt_name
1366 s ∈ a.short_opt_name || push!(a.short_opt_name, s)
1367 end
1368 for al in oa.cmd_aliases
1369 al ∈ a.cmd_aliases || push!(a.cmd_aliases, al)
1370 end
1371 a.group = oa.group # note: the group may not be present yet, but it will be
1372 # added later
1373 push!(oids, ioa)
1374 end
1375 end
1376 # we return the merged ofields indices, since we still need to use them for overriding options
1377 # before we actually remove them
1378 return oids
1379 end

1391 """
1392 import_settings!(settings, other_settings [,args_only])
1393
1394 Imports `other_settings` into `settings`, where both are [`ArgParseSettings`](@ref) objects. If
1395 `args_only` is `true` (this is the default), only the argument table will be imported; otherwise,
1396 the default argument group will also be imported, and all general settings except `prog`,
1397 `description`, `epilog`, `usage` and `version`.
1398
1399 Sub-settings associated with commands will also be imported recursively; the `args_only` setting
1400 applies to those as well. If there are common commands, their sub-settings will be merged.
1401
1402 While importing, conflicts may arise: if `settings.error_on_conflict` is `true`, this will result in
1403 an error, otherwise conflicts will be resolved in favor of `other_settings` (see the
1404 [Conflicts and overrides](@ref) section for a detailed discussion of how conflicts are handled).
1405
1406 Argument groups will also be imported; if two groups in `settings` and `other_settings` match, they
1407 are merged (groups match either by name, or, if unnamed, by their description).
1408
1409 Note that the import will have effect immediately: any subsequent modification of `other_settings`
1410 will not have any effect on `settings`.
1411
1412 This function can be used at any time.
1413 """
1414 function import_settings!(settings::ArgParseSettings,
1415 other::ArgParseSettings;
1416 args_only::Bool = true)
1417 check_settings_are_compatible(settings, other)
1418
1419 fields = settings.args_table.fields
1420 ofields = deepcopy(other.args_table.fields)
1421 merged_oids = merge_commands!(fields, ofields)
1422 if !settings.error_on_conflict
1423 for a in ofields
1424 override_duplicates!(fields, a)
1425 end
1426 for (subk, subs) in other.args_table.subsettings
1427 override_conflicts_with_commands!(settings, subk)
1428 end
1429 end
1430 while !isempty(merged_oids)
1431 splice!(ofields, pop!(merged_oids))
1432 end
1433 append!(fields, ofields)
1434 for oag in other.args_groups
1435 skip = false
1436 for ag in settings.args_groups
1437 # TODO: merge groups in some cases
1438 if oag.name == ag.name
1439 skip = true
1440 break
1441 end
1442 end
1443 skip && continue
1444 push!(settings.args_groups, deepcopy(oag))
1445 end
1446
1447 fix_commands_fields!(fields)
1448
1449 if !args_only
1450 settings.add_help = other.add_help
1451 settings.add_version = other.add_version
1452 settings.error_on_conflict = other.error_on_conflict
1453 settings.suppress_warnings = other.suppress_warnings
1454 settings.exc_handler = other.exc_handler
1455 settings.allow_ambiguous_opts = other.allow_ambiguous_opts
1456 settings.commands_are_required = other.commands_are_required
1457 settings.default_group = other.default_group
1458 settings.preformatted_description = other.preformatted_description
1459 settings.preformatted_epilog = other.preformatted_epilog
1460 settings.fromfile_prefix_chars = other.fromfile_prefix_chars
1461 settings.autofix_names = other.autofix_names
1462 settings.exit_after_help = other.exit_after_help
1463 end
1464 for (subk, subs) in other.args_table.subsettings
1465 cmd_prog_hint = ""
1466 for oa in other.args_table.fields
1467 if is_cmd(oa) && oa.constant == subk
1468 cmd_prog_hint = get_cmd_prog_hint(oa)
1469 break
1470 end
1471 end
1472 if !haskey(settings, subk)
1473 add_command!(settings, subk, cmd_prog_hint, !settings.error_on_conflict)
1474 elseif !isempty(cmd_prog_hint)
1475 settings[subk].prog = "$(settings.prog) $cmd_prog_hint"
1476 end
1477 import_settings!(settings[subk], subs, args_only=args_only)
1478 end
1479 return settings
1480 end

1502 function project_version(filename::AbstractString)::String
1503 re = r"^version\s*=\s*\"(.*)\"\s*$"
1504 for line in eachline(filename)
1505 if startswith(line, "[")
1506 break
1507 end
1508 if !occursin(re, line)
1509 continue
1510 end
1511 return match(re, line)[1]
1512 end
