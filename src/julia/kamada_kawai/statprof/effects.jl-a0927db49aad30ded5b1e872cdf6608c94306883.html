3
4 Represents computational effects of a method call.
5
6 The effects are a composition of different effect bits that represent some program property
7 of the method being analyzed. They are represented as `Bool` or `UInt8` bits with the
8 following meanings:
9 - `consistent::UInt8`:
10 * `ALWAYS_TRUE`: this method is guaranteed to return or terminate consistently.
11 * `ALWAYS_FALSE`: this method may be not return or terminate consistently, and there is
12 no need for further analysis with respect to this effect property as this conclusion
13 will not be refined anyway.
14 * `CONSISTENT_IF_NOTRETURNED`: the `:consistent`-cy of this method can later be refined to
15 `ALWAYS_TRUE` in a case when the return value of this method never involves newly
16 allocated mutable objects.
17 * `CONSISTENT_IF_INACCESSIBLEMEMONLY`: the `:consistent`-cy of this method can later be
18 refined to `ALWAYS_TRUE` in a case when `:inaccessiblememonly` is proven.
19 - `effect_free::UInt8`:
20 * `ALWAYS_TRUE`: this method is free from externally semantically visible side effects.
21 * `ALWAYS_FALSE`: this method may not be free from externally semantically visible side effects, and there is
22 no need for further analysis with respect to this effect property as this conclusion
23 will not be refined anyway.
24 * `EFFECT_FREE_IF_INACCESSIBLEMEMONLY`: the `:effect-free`-ness of this method can later be
25 refined to `ALWAYS_TRUE` in a case when `:inaccessiblememonly` is proven.
26 - `nothrow::Bool`: this method is guaranteed to not throw an exception.
27 If the execution of this method may raise `MethodError`s and similar exceptions, then
28 the method is not considered as `:nothrow`.
29 However, note that environment-dependent errors like `StackOverflowError` or `InterruptException`
30 are not modeled by this effect and thus a method that may result in `StackOverflowError`
31 does not necessarily need to taint `:nothrow` (although it should usually taint `:terminates` too).
32 - `terminates::Bool`: this method is guaranteed to terminate.
33 - `notaskstate::Bool`: this method does not access any state bound to the current
34 task and may thus be moved to a different task without changing observable
35 behavior. Note that this currently implies that `noyield` as well, since
36 yielding modifies the state of the current task, though this may be split
37 in the future.
38 - `inaccessiblememonly::UInt8`:
39 * `ALWAYS_TRUE`: this method does not access or modify externally accessible mutable memory.
40 This state corresponds to LLVM's `inaccessiblememonly` function attribute.
41 * `ALWAYS_FALSE`: this method may access or modify externally accessible mutable memory.
42 * `INACCESSIBLEMEM_OR_ARGMEMONLY`: this method does not access or modify externally accessible mutable memory,
43 except that it may access or modify mutable memory pointed to by its call arguments.
44 This may later be refined to `ALWAYS_TRUE` in a case when call arguments are known to be immutable.
45 This state corresponds to LLVM's `inaccessiblemem_or_argmemonly` function attribute.
46 - `noub::UInt8`:
47 * `ALWAYS_TRUE`: this method is guaranteed to not execute any undefined behavior (for any input).
48 * `ALWAYS_FALSE`: this method may execute undefined behavior.
49 * `NOUB_IF_NOINBOUNDS`: this method is guaranteed to not execute any undefined behavior
50 if the caller does not set nor propagate the `@inbounds` context.
51 Note that undefined behavior may technically cause the method to violate any other effect
52 assertions (such as `:consistent` or `:effect_free`) as well, but we do not model this,
53 and they assume the absence of undefined behavior.
54 - `nonoverlayed::UInt8`:
55 * `ALWAYS_TRUE`: this method is guaranteed to not invoke any methods that defined in an
56 [overlayed method table](@ref OverlayMethodTable).
57 * `CONSISTENT_OVERLAY`: this method may invoke overlayed methods, but all such overlayed
58 methods are `:consistent` with their non-overlayed original counterparts
59 (see [`Base.@assume_effects`](@ref) for the exact definition of `:consistenct`-cy).
60 * `ALWAYS_FALSE`: this method may invoke overlayed methods.
61 - `nortcall::Bool`: this method does not call `Core.Compiler.return_type`,
62 and it is guaranteed that any other methods this method might call also do not call
63 `Core.Compiler.return_type`.
64
65 Note that the representations above are just internal implementation details and thus likely
66 to change in the future. See [`Base.@assume_effects`](@ref) for more detailed explanation
67 on the definitions of these properties.
68
69 Along the abstract interpretation, `Effects` at each statement are analyzed locally and they
70 are merged into the single global `Effects` that represents the entire effects of the
71 analyzed method (see the implementation of `merge_effects!`). Each effect property is
72 initialized with `ALWAYS_TRUE`/`true` and then transitioned towards `ALWAYS_FALSE`/`false`.
73 Note that within the current flow-insensitive analysis design, effects detected by local
74 analysis on each statement usually taint the global conclusion conservatively.
75
76 ## Key for `show` output of Effects:
77
78 The output represents the state of different effect properties in the following order:
79
80 1. `consistent` (`c`):
81 - `+c` (green): `ALWAYS_TRUE`
82 - `-c` (red): `ALWAYS_FALSE`
83 - `?c` (yellow): `CONSISTENT_IF_NOTRETURNED` and/or `CONSISTENT_IF_INACCESSIBLEMEMONLY`
84 2. `effect_free` (`e`):
85 - `+e` (green): `ALWAYS_TRUE`
86 - `-e` (red): `ALWAYS_FALSE`
87 - `?e` (yellow): `EFFECT_FREE_IF_INACCESSIBLEMEMONLY`
88 3. `nothrow` (`n`):
89 - `+n` (green): `true`
90 - `-n` (red): `false`
91 4. `terminates` (`t`):
92 - `+t` (green): `true`
93 - `-t` (red): `false`
94 5. `notaskstate` (`s`):
95 - `+s` (green): `true`
96 - `-s` (red): `false`
97 6. `inaccessiblememonly` (`m`):
98 - `+m` (green): `ALWAYS_TRUE`
99 - `-m` (red): `ALWAYS_FALSE`
100 - `?m` (yellow): `INACCESSIBLEMEM_OR_ARGMEMONLY`
101 7. `noub` (`u`):
102 - `+u` (green): `true`
103 - `-u` (red): `false`
104 - `?u` (yellow): `NOUB_IF_NOINBOUNDS`
105 8. `:nonoverlayed` (`o`):
106 - `+o` (green): `ALWAYS_TRUE`
107 - `-o` (red): `ALWAYS_FALSE`
108 - `?o` (yellow): `CONSISTENT_OVERLAY`
109 9. `:nortcall` (`r`):
110 - `+r` (green): `true`
111 - `-r` (red): `false`
